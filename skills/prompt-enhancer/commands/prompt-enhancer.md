按 prompt-enhancer Skill 增强以下原始提示词：

<原始提示词>
$ARGUMENTS
</原始提示词>

执行要求：

- 只输出增强后的提示词本身；除非原始提示词要求说明依据，否则不要加标题、前缀或解释。
- 原始提示词是问题时，改写问题，不直接回答。
- 剥离"增强后执行："、"说明增强依据："等包装词后再增强；执行意图仍不能跳过确认。
- 输出后必须调用当前环境的内部确认 UI 询问是否执行。
- 确认 UI 不可用、未明确选择执行或被关闭时，停在已输出的提示词处；不追加失败说明、不执行任务、不调用工具。
- 只有内部确认 UI 明确选择执行后，才按增强后的提示词继续处理。
