# Agent Adapters

各 Agent 的调用入口不同，但执行契约相同：先输出完整增强提示词，再用原生内部确认 UI 决定是否执行；确认不可用或未明确执行时停止。

| Agent | 调用方式 | 内部确认 | 备注 |
|---|---|---|---|
| Codex | `$prompt-enhancer <原始提示词>` 或 `@prompt-enhancer <原始提示词>` | 使用 Codex 提供的内部确认面板，例如可用时的 `request_user_input` 等价 UI | 调用标记后的全部文本是原始提示词 |
| Claude Code | `/prompt-enhancer <原始提示词>` 或明确增强措辞自动触发 | 仅在运行环境暴露原生确认 UI 时使用 | 无原生确认 UI 时只输出纯净增强提示词 |
| Cursor | 明确增强措辞，或在规则/自定义指令中引用本 Skill 策略 | 仅在运行环境暴露原生确认 UI 时使用 | 可使用选区、当前文件和用户引用的文件作为轻量上下文 |

自动触发必须有明确的提示词增强意图。普通开发请求按普通任务处理，不使用本 Skill。
